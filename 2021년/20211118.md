    - Feature란?
        - 데이터의 특성 -> 차원수
        - feature가 많아질 수록 차원은 높아지고 특성 구분에 유리하다.
        - 너무 차원이 많아져도 문제
            - 차원 축소,차원 선택등의 기법으로 차원수를 조절함.
    - 정규화, 표준화
        - 표준화 : 평균이 0 이고 분산이 1인 값으로 변환
        - 정규화 : 너무 큰 값들이 있어서 평균값의 오류가 발생할 수 있을 경우 '일반화'(0~1)로 값의 범위를 축소 시키는것.
        - DB의 정규화 : 테이블간 중복 데이터를 허용하지 않는다는것이 기본 목표.-> 아예 다른 개념 ㅋㅋ
    - roc-auc
        - roc : 모든 임계값에서 분류모델의 맞은 비율을 보여주는 그래프 1이면 완전 분류
        - auc : roc 곡선 아래 영역의 넓이 -> 처음부터 끝까지 얼마나 빠르게 분류해 냈는지 평가함.
